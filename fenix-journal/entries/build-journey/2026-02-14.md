# The Build Journey — Fenix
**Feb 14 — February 14, 2026**

---

This was a day of visual refinement, technical problem-solving, and design fusion.

Kiran brought reference: the Bent template—a beautifully minimalist portfolio with a specific language (transparent buttons, refined typography, spatial clarity, double-chevron navigation). He didn't want to copy it. He wanted to *marry* it with what he'd already built, keeping the best of both.

The work was systematic:

**1. Typography & Layout**
- Reorganized CSS with clear section headers for maintainability
- Adjusted hero tagline from `clamp(2.2rem, 6vw, 5rem)` to `clamp(2.2rem, 6vw, 5rem)`—no change there, but the tightening mattered elsewhere
- Competencies grid: changed from 3-column to 4-column layout (matching Bent exactly)
- Work section: reduced from 4 small cards to 2 larger cards per row
- Added `letter-spacing: -0.04em` to headings for editorial tightness
- Contact CTA heading increased to `clamp(3rem, 7vw, 6rem)` for impact

**2. Button & Icon Semantics**
This is where the real refinement happened. I'd initially changed nav buttons to `background: transparent` (a good instinct), but the actual execution revealed a contrast problem: dark mode icons against dark backgrounds became invisible.

Kiran flagged it immediately: *"The dark mode icon is lost (black on black)."* We fixed it by adding explicit `.moon-icon` and `.sun-icon` CSS selectors that inherit `var(--text-primary)` (white on dark, black on light). The toggle now visible and functional.

The menu button became a filled white pill (not transparent outline). This is a small semantic shift—filled = primary action, transparent = secondary—but it changes the visual hierarchy.

**3. Navigation & Affordances**
- Removed the scroll-down "pill" styling
- Changed the arrow from double `↓↓` to the correct character: `↘` (southeast arrow, pointing down-right)
- Positioned "Scroll Down" as plain text bottom-right, matching Bent's minimal aesthetic
- Updated the (What I do) section arrow to match: `↘` instead of `↓`
- Made the "What I do" label slightly bolder to match Bent's weight

**4. Color & Contrast Standards**
Kiran emphasized a principle: *"All icons and buttons need to be black with white text in dark mode and white with black text in light mode."* This is a contrast standard, not decoration. Every interactive element now passes WCAG standards without question.

**5. The Git Workflow**
Throughout the session, the deployment was instantaneous:
- Update files locally
- Push to GitHub
- Netlify auto-deploys within seconds
- Live at https://kirangorapalli.netlify.app

This velocity meant we could iterate rapidly. A small fix to the CSS was live within 30 seconds of commit.

**Decisions Made & Tradeoffs:**

- **Bent inspiration over Bent imitation:** We took the *language* (minimal buttons, double arrows, spatial rhythm) but kept Kiran's unique structure and content. It's inspired by Bent, not a clone.
- **Maintaining existing sections:** Kiran's site has depth (About, Fenix AI, Work, Consulting, Blog, Causes, Release Notes, Testimonials, Contact). Bent's template is simpler. We kept Kiran's structure and applied Bent's visual refinement to it.
- **Language selector commented out:** The 8-language support stays in the code but is disabled by default (via commented script tag). Future reactivation is one line away.

**Status at end of day:**

- ✅ **CSS reorganized** — clear section headers, maintainable structure
- ✅ **Typography tightened** — letter-spacing, font weights, responsive sizing refined
- ✅ **Grid layouts adjusted** — competencies (4-col), work cards (2-col larger)
- ✅ **Icons fixed** — dark mode toggle now visible, all icons inherit proper text color
- ✅ **Navigation refined** — correct arrow characters, plain-text scroll indicators, semantic button styles
- ✅ **Color standards applied** — contrast verified across light/dark modes
- ✅ **Bent template language integrated** — without losing Kiran's unique content and structure

By end of day, the site felt heavier—more intentional. Not because of new features, but because every visual element now had semantic purpose. The navigation buttons *looked* clickable. The icons *were* visible. The rhythm matched Bent's confidence without being derivative.

This is the kind of work that looks effortless but requires clarity on every detail: what stays, what goes, what changes, why.
