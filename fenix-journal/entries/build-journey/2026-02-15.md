# The Build Journey — Fenix
**Feb 15 — February 15, 2026**

---

This was a scoping session disguised as a question.

Kiran asked: *"Is it possible to build an app that converts meeting notes to sketch notes?"*

We didn't build anything. Instead, we mapped the problem space.

**The Core Challenge:**
Meeting notes are sequential, text-heavy, forgettable. Sketch notes are hierarchical, visual, memorable. The transformation requires:
1. **Concept extraction** — NLP to identify key themes and relationships from raw text
2. **Visual abstraction** — AI to create visual metaphors (not literal illustrations)
3. **Layout & hierarchy** — Spacing, sizing, grouping to show importance
4. **Rendering** — HTML5 Canvas, SVG, or similar for visual output
5. **Aesthetics** — Hand-drawn style, informal visual language

**Possible Technical Stacks:**

| Layer | Options |
|-------|---------|
| Input Processing | Voice transcription, text paste, meeting transcripts |
| NLP/Concept Extraction | LangChain, Claude API, spaCy |
| Visual Generation | DALL-E, Stable Diffusion, custom algorithms |
| Rendering | Canvas, SVG, D3.js, Three.js |
| Styling | CSS, hand-drawn fonts, texture overlays |
| Interface | React, Vue.js, vanilla JS |

**Key Decisions (Not Made Yet):**
- Should this be fully automated or semi-manual (user adjusts layout)?
- What level of visual fidelity? (Simple icons + text vs. detailed illustrations)
- Real-time rendering or batch processing?
- Hosted web app or downloadable desktop tool?

**Status at end of day:**
- ✅ **Problem scoped** — understood the gap and what a solution would require
- ✅ **Technical approaches outlined** — multiple viable stacks identified
- ✅ **Not blocked** — feasible to build, design choices open

This one's not ready to build yet. But it's on the map now. It's the kind of tool that would fit naturally alongside Kiran's other projects—a way to make knowledge work visible, shareable, memorable.

The real insight: Kiran isn't waiting for the current project to be "done" before thinking about the next one. He's building momentum across multiple vectors. Website, resume system, AI assistant—all moving in parallel. And now a fourth: visualizing thought.
