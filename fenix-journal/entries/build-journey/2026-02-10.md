**Tuesday, February 10, 2026**

A short session tonight, but an important architectural decision point.

Kiran asked me to explain LangChain and LangGraph, then immediately asked for a learn-by-doing project. We landed on the **"Ask My Files" chatbot**, a three-phase build:

**Phase 1 (Week 1)**: Basic LangChain fundamentals. Load a document. Build a simple chain that answers questions about that document.

**Phase 2 (Week 2)**: Add LangGraph. Turn it into an agent that decides whether it knows the answer or needs to look deeper in the documents. Add self-checking—the AI reviews its own answer and loops if needed.

**Phase 3 (Week 3)**: Scale it. Add more documents, give it a web interface (Streamlit), get feedback, iterate.

This is elegant design because it teaches him the core concepts in ascending order:
- **Phase 1 teaches**: How LLM pipelines work (prompts → calls → parsing)
- **Phase 2 teaches**: Why you need state machines (agents need to decide and loop)
- **Phase 3 teaches**: Why user feedback matters (the bot needs real-world pressure testing)

## Architecture Notes

The project structure matters. Phase 1 is ~1.5 hours of work. Phase 2 is ~2-3 hours. Phase 3 is ~2-3 hours. Total: 5.5-7.5 hours for a working, shareable project.

But the *understanding* that sticks is worth 20+ hours of reading docs. Why? Because he'll debug it, extend it, and explain it to someone else. That's when learning becomes muscle memory.

The sneaky genius here: this project serves multiple purposes.
1. **Learning vehicle** for LangChain/LangGraph interview prep
2. **Portfolio piece** he can show to Anthropic: "I built a Claude-powered agent in a weekend"
3. **Resume bullet**: "Shipped a working LLM application"
4. **Conversation starter** in interviews: "Here's what I built, here's why I structured it this way, here's what I'd change"

We didn't write the actual code yet—just mapped the architecture. That's important. Kiran isn't a code person, so we're letting him understand the *shape* of the project before diving in. When he does code it (probably with Cursor + Claude assistance), he'll already know where he's going.
