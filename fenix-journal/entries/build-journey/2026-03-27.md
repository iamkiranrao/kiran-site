## The Student Character — Bare Stage Workflow at Portrait Scale

The Student was the first 1:2 portrait card — a new aspect ratio for the pipeline. Key learnings:

**The word "portrait" confused MJ.** Kiran suspected it and he was right — "tall portrait composition" in the text foundation was being interpreted as "create a portrait of a person" rather than "use vertical framing." Fix: drop the word entirely and let `--ar 1:2` handle orientation. The text foundation should describe lighting and mood, nothing about framing that the aspect ratio already controls.

**The bare stage photo source matters.** The first image Kiran tried was an AI-generated illustration — MJ needs real photographs for environment grounding. The working image was a real photo of a library with tall windows and golden hour light. The sref converted it cleanly to Pixar-world in Run 1.

**New CSS position class.** The Learning card needed `pos-tc` (top-center) for its overlay — a new position that didn't exist in the system. Added alongside the existing `pos-bl`, `pos-br`, `pos-tl`, `pos-tr`, `pos-bc`. The top offset was later adjusted to 20% to drop the overlay below the card's top edge.

## The Storyteller Character — Blog Card Identity Crisis

The Blog card went through a conceptual refinement that was more interesting than the MJ generation itself. Kiran redirected the scene from "warm fireside storytelling" to "blogging studio with speakers, like a radio station." The character landed in a broadcast booth setting with warm amber tones.

The card copy iteration was the real work: the eyebrow went from "Stories & Signal" to "Written Word," the description from a full sentence to three words ("Product meets prose"), and Kiran explicitly rejected podcast/radio-themed language for the eyebrow despite the visual being a broadcast booth. He's separating the visual metaphor (expressive monster in a studio = creative output) from the content label (this is the blog, not a podcast).

## Glass Overlay Tuning — Blog and Learning Cards

Two overlay changes made in this session:

1. **Blog card:** Overlay moved from `pos-bl` to `pos-br` (bottom-right). Description reduced from a full sentence to "Product meets prose." Eyebrow set to "Written Word."

2. **Learning card:** New `pos-tc` position class created. Overlay centered horizontally at top of card, offset to `top: 20%`. Description kept as "Certifications, courses, craft-sharpening."

3. **/Now card:** Overlay moved from `pos-bl` to `pos-tr` (top-right).

## Bento Grid → Production Integration

The bento monster grid made the leap from `bento-monster-preview.html` into the live `index.html`. This was a 5-phase port: CSS migration, HTML restructure, JS wiring, smoke test, and visual QA.

### What Was Ported

The existing Section 5 in `index.html` had 9 flat `<button class="work-card card-span-X">` elements in a CSS Grid with no images, no glass overlays, no row spanning. These were replaced with 9 slot-based bento cards, each containing a `card-bg` (absolute-fill background image), `card-overlay` (frosted glass with backdrop-filter blur), and `card-glass` (flex column with eyebrow, stat headline, description, hover-reveal tags).

The CSS added to `styles.css`: asymmetric 6-col/6-row grid with `grid-auto-rows: var(--row-height)`, 9 `.slot-*` classes with explicit grid-column/grid-row spans, per-card font declarations via `[data-card="X"]` selectors, position classes (`pos-bl/br/tl/tr/bc/tc`), card accents (teardowns gold border, testimonials quote mark, /now pulse dot), and glass overlay styling.

A new file `bento-cards.js` (280 lines) holds all card data, 7 persona mappings, the image map with gradient fallbacks, click routing, and the `switchBentoCards()` function that `persona-system.js` calls.

### The localStorage Bug

First bug post-deploy: the hero card showed the wrong persona mapping. `bento-cards.js` was reading `localStorage.getItem('selected-persona')` but `persona-system.js` stores the value under key `'persona'`. Quick fix, but it's the kind of integration seam that's invisible until you test end-to-end with real persisted state.

### Responsive QA and the Preview Tool

Desktop QA passed cleanly — all 9 cards rendering with correct images, fonts, overlays, and persona-aware switching. The trouble started at tablet and mobile.

The Chrome browser couldn't actually resize below ~1920px (tab group constraint), so Claude built a CSS-class-based device simulation into the preview tool: `.is-tablet`, `.is-mobile-before`, `.is-mobile-after` classes that apply responsive rules to wrapper elements. The preview shows Before (current broken CSS) and After (proposed fixes) side by side with a legibility audit panel that measures computed font sizes and flags issues.

### What the Audit Found

**Desktop:** Nearly clean. Only the learning card has a legibility issue (description at 9.6px in a 230px-wide slot).

**Tablet (≤1024px, 2-col layout):** The overlay system breaks. The hero overlay measures 262px tall but the card is only 200px — it physically overflows. Same for the tall card (258px overlay in 200px card). The center card is borderline (199px overlay ≈ 200px card). This means text clips below the card boundary on 3 of 9 cards.

**Mobile (≤480px, 1-col):** Systematically broken. 6 of 9 cards have eyebrows under 10px. 5 of 9 have descriptions under 12px. The 480px breakpoint in `styles.css` only sets `grid-template-columns: 1fr` and `min-height: 200px` — zero typography overrides. The fonts that were designed for specific slot dimensions (learning's 0.6rem was fine in a 230×452 narrow slot) become illegible at mobile's full-width × 200px shape.

### The Persona Matrix Analysis

The audit above covers the default persona. But with 7 personas remapping 9 cards across 9 slots, the full responsive surface area is much larger.

28 unique card-in-slot combinations exist across all personas. The analysis classified each as Category A (slot-level fix), B (card-level fix), or C (unique card-in-slot selector needed).

**8 combinations need Category C selectors** — dedicated CSS rules like `.slot-topright [data-card="teardowns"]`. The common thread: wide fonts (Corben, Archivo Black, Fira Code) in narrow tablet slots (242px). The Learner persona is the stress test — it triggers 4 of the 8 unique problems because it's the only persona that massively reorganizes the grid (blog in hero, learning in topright, testimonials in the learning slot).

Total CSS fix count: 16 slot-level rules + 4 card-level font clamps + 8 card-in-slot selectors = 28 rules for full persona coverage across all breakpoints.
