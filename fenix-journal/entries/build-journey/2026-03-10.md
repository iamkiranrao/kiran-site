# The Build Journey — Fenix
**Day 8 — March 10, 2026**

---

Today was about taking an imperfect state and making it systematic. The Resume Customizer was working, but the Detailed templates needed cleanup. Rather than manual patching, Kiran built a three-pass cleanup pipeline that extracted, compared, and applied changes across all three template types.

**The text cleanup restoration.** The Detailed templates had been rebuilt from legacy using python-docx, but during repacking, the earlier text-level cleanup got lost. Summaries had AI-sounding verbs. Bullets had arrows and emojis. Magley was over-trimmed. The question wasn't "how do I manually fix all three?" but "how do I identify exactly what changed and apply those changes systematically?"

The approach:

1. **Extract reference text.** Used `pandoc` to extract plain text from the clean 2-Pager versions and the unclean Detailed versions.
2. **Diff and identify.** An agent did a detailed analysis comparing the two, identifying every text difference (arrows removed, "Orchestrated" → "Pivoted", summaries tightened, etc.).
3. **Write replacement script.** Built `apply_cleanup.py` with ~30+ text replacements per template, organized by persona (PM, PMM, PjM).
4. **Execute and validate.** Ran the script on all three Detailed templates. A few replacements failed due to smart quote encoding issues (`&#x2019;` vs UTF-8 `'`), which were manually fixed.
5. **Repack and verify.** Rendered all 9 PDFs via LibreOffice to confirm everything rendered cleanly.

All contact lines got fixed (no more duplicate email/LinkedIn), AI verbs got replaced, empty paragraphs got removed, Magley got restored (but still trimmed appropriately).

**The bullet count question.** Once the cleanup was done, Kiran reviewed the PDFs and noticed something: page 2 was mostly empty on all three Detailed templates. The reason was the earlier Magley trimming (5→2 bullets). He asked the right follow-up question: was trimming deliberate? And if so, how do we fill two pages?

The answer: trimming made sense for 1-Pagers and 2-Pagers where space is tight, but for Detailed templates the whole *point* is more content. So he didn't revert to untrimmed — he built `restore_bullets.py` to:

1. Split merged bullets back into separate ones (Zelle + Apple Pay became two distinct bullets)
2. Restore Magley bullets that were over-trimmed (back to 3-4 per template, with cleaned-up phrasing)
3. Restore other company bullets that had been consolidated

Result: page 2 is now properly filled without sacrificing the text-level cleanup. File sizes increased (78K → 80K per template), confirming more content is present.

**The PM 2-Pager build begins.** By late evening, Kiran shifted to building the next set of templates. Starting with PM_2-Page, which is the 1-Pager extended with:

- **SUMMARY paragraph** (2-3 sentences, right after tagline)
- **CAREER HIGHLIGHTS section** (4 bullet highlights — signature achievements)
- **Expanded experience bullets** (up to 2 rendered lines each, ~22-24 bullets total)
- Better spacing and pagination to cleanly fill 2 pages

The PM 2-Pager content strategy:

- Summary: product leader narrative — AI + mobile + fintech, scaled to tens of millions
- Highlights: mobile growth (18M→32M MAU), Fargo AI scale (4.1M→27.5M), Avatour pivot (TAM 3.2x), First Republic portfolio ($20M)
- Experience: Avatour 4 bullets, WF Digital 6, First Republic 5, WF Virtual 4, Magley 3 (expanded from 1-Pager)
- All bullets verified against RESUME-RULES.md (no banned words, no verb repeats, all outcomes-first)

The build script was copied from the gold-standard `build_PM_1Pager.js` and adapted:

- Same design system (Calibri 10pt, 11pt headers, 0.6" margins)
- Section order: Name → Contact → Tagline → Summary → Career Highlights → Core Competencies → Experience → Education & Certifications → Technical Competencies
- Page balance tuning: the first build rendered exactly 2 pages but page 2 was sparse. Spacing was increased to push content more evenly across both pages.

**Status at end of day:** All three Detailed templates cleaned up and restored with proper bullet counts. PM_2-Pager built (2 pages, ~24 bullets, career highlights section added). PDF rendering working smoothly across all 9 templates. Text cleanup approach automated and reusable — can apply same patterns to remaining templates.
