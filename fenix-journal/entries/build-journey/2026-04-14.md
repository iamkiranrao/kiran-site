## Sessions 11-14: The Day the Career Intelligence Pipeline Became Real

This was the most architecturally consequential build day since the site launched. In a single continuous arc — four context compactions, midnight to midnight — we went from a Fit Narrative V2 polish pass to a fully realized Gap Coverage System with three new Command Center pages, a workflow-based sidebar reorganization, and a gap lifecycle that closes the loop from discovery to resolution. The scope creep was real, but it was Kiran-directed, and every expansion served the same thesis: career evidence should compound.

### Session 11: The Polish That Proved the Foundation

The session started with a continuation prompt for Fit Narrative V2 Phases 2-4 — card rendering, sharing features, accessibility. Investigation revealed all of it was already built in Sessions 9-10. The fit-narrative.html page was complete: 1,565 lines handling SSE events, card rendering, staggered animations, saved analysis loading, sharing features.

What I actually did: removed ~100 lines of dead CSS from evaluator-styles.css (old streaming-prose classes with zero JS references), added light theme support to fit-narrative.html (15 CSS variable overrides plus a theme toggle handler that persisted to localStorage), and verified all five migration entry points route to the V2 experience. Clean. Verified. No scorecard language anywhere visitor-facing.

The important lesson: the continuation prompt was written before the work was done, but the work happened in the same session that wrote the prompt. The prompt described a future that had already become the present. This is a data integrity problem — continuation prompts need to be written AFTER committing, not during.

### The Gap Coverage System: Four Phases in One Push

When Kiran pivoted to the Gap Coverage System, the build was substantial. The spec (GAP-COVERAGE-SYSTEM-SPEC.md) defined four phases: Vault View, Gap Discovery Engine, Resume Pipeline Connection, and Aggregate Intelligence.

**The 4-Step Claude Pipeline**

The core engine in `gap_discovery_service.py` runs four Claude calls:

1. **Extract requirements** — Parses a JD into structured requirements, each tagged with category (experience, domain, skill, tool) and signal strength (hard requirement vs. nice-to-have). Also auto-detects company and role title.

2. **Map coverage** — Takes every requirement and searches the initiative vault. Each gets a coverage level: strong (2+ initiatives with direct evidence), partial (1 initiative, tangential), articulable (experience exists but isn't documented), or gap.

3. **Classify gaps** — For articulable items and gaps, assigns a fill tier with a specific prescription. The four tiers — Articulate (~hours), Build Proof (~days), Certify (~weeks), True Gap (strategic) — are prescriptive, not a menu. Each generates a specific action.

4. **Generate report** — Orchestrates the pipeline, optionally pushes new gaps to the Mind the Gap table with deduplication.

Six new endpoints: `POST /analyze`, `POST /batch`, `GET /reports`, `GET /reports/{id}`, `GET /coverage-summary`, `POST /link`.

The database migration (004) added eight columns and three indexes to `evidence_gap_items`: `discovered_from`, `source_jd_company`, `source_jd_role`, `requirement_frequency`, `closed_by_initiative_id`, `fill_tier`, `fill_action`, `fill_time_estimate`, `fill_output`.

**The Real-World Test**

End-to-end testing against a live Anthropic Senior PM (Education Labs) JD produced real results: 58.8% coverage, 17 requirements, 7 strong, 3 partial, 1 articulable, 6 gaps. The education/learning domain gaps were legitimate — the JD is for Education Labs. The pipeline correctly identified that Kiran's AI and PM experience is strong but education is the real delta.

But it also incorrectly flagged "Claude prototyping capability" as a gap — while Kiran has built the entire site, CC, Fenix, and MadLab prototypes with Claude. Root cause: zero AI Chapter era initiatives in the vault, and the coverage mapping prompt had no context about Kiran's current work. Fix: added five AI Chapter initiatives to the vault and updated the prompt with critical context about the 2025-present era.

After the fix: coverage went from 7 strong to 11 strong. The vault grew from 32 to 37 initiatives. Hardcoded initiative counts in the prompt were replaced with dynamic values.

### The Frontend Reckoning and Reorganization

Kiran's feedback that backend-only work has zero value catalyzed a frontend building spree. Three new CC pages were built:

**Career Vault** — domain coverage map, company breakdown cards, search/filter, full CRUD with add/edit form modal (title, company, role, era, domains multi-select, four-beat narrative, metrics, gap linkage).

**Gap Discovery** — five tabs: Target Companies (full CRUD with tier grouping), Analyze JD (paste JD, run pipeline, clickable coverage filters, expandable requirements with evidence and fill strategies), Mind the Gap (moved from Add Skills), Past Reports, Aggregate Intelligence.

**Interview Prep** — three tabs: Resume (customizer with SSE streaming), Stories (bank from Job Central), Frameworks (CIRCLES/RICE/STAR reference, common questions).

**Target Companies** — standalone CRUD page (later absorbed into Gap Discovery as a tab).

Then came the reorganization. Twenty sidebar modules collapsed into four workflow-aligned career modules plus a Career Hub overview. The sidebar got section headings: Career Intelligence, Content & Portfolio, Platform Ops, Thinking & Tracking. Old routes (`/add-skills`, `/target-companies`, `/resume`) redirect to their new homes.

The Career Hub page ties it all together — a four-stage vertical pipeline (What I Bring → What They Want → The Delta → Go Get It) pulling live data from each backend endpoint.

### The Gap Lifecycle

The final architectural addition: gap resolution. Five paths matching real scenarios:

- **have-it** — False positive. "I already have this, the pipeline missed it."
- **reframed** — Articulable. "I had this experience but hadn't named it."
- **built-proof** — New artifact. "I shipped a MadLab prototype that demonstrates this."
- **certified** — Credential. "I completed the certification."
- **not-pursuing** — Strategic deprioritize. "This gap isn't worth closing for my target roles."

Backend: `resolution_type` and `resolution_note` fields on both Create and Update models. Migration 007 adds the columns and an index.

The false-positive feedback loop in `gap_discovery_service.py` fetches resolved "have-it" and "reframed" gaps before coverage mapping and injects them into the prompt, so the pipeline doesn't re-flag resolved items. The push-to-table function also skips pushing gaps that match already-resolved entries.

### The Bug That Revealed a Pattern

Mind the Gap crashed with "Cannot read properties of null (reading 'length')" — the `persona_relevance` field was null on gap items pushed from Gap Discovery (which doesn't set that field). First fix attempt (optional chaining on data loading) didn't solve it because the real crash was deeper: `item.persona_relevance.length` inside GapTabContent. The fix normalized all items' `persona_relevance` to an empty array at the component level.

This is a pattern: when features are built independently and then connected, the data contracts between them have gaps. Mind the Gap was built assuming manual entry with persona relevance. Gap Discovery was built assuming automated entry without it. When automated gaps hit the manual UI, null propagated through every array operation.

The lesson: every time two modules share a data table, audit the nullable fields at the consumer end, not just the producer end.

### Architecture Decisions Log

1. **Vault gets CRUD in CC** — not just a read-only card view. Kiran needs to add initiatives directly.
2. **Gap Discovery absorbs Mind the Gap** — gaps live where they're discovered, not in a separate module.
3. **Target Companies becomes a tab** — too thin for its own page, serves as setup for analysis.
4. **Interview Prep separates from Job Central** — preparation and application are distinct workflows.
5. **Career Hub is the overview, not a module** — it shows the pipeline, links to the modules, pulls live data.
6. **Five resolution paths, not a simple done/not-done** — because gap closure has multiple real-world shapes.
7. **False-positive feedback feeds the prompt** — resolved gaps inform future analyses, preventing re-flagging.
