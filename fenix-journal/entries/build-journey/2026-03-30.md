# The Build Journey — Career Highlights Infographic Strategy
**March 30, 2026**

---

## What Happened: A Strategy Session, Not a Build Session

No code was shipped in production today. One change went live — removing the access gate on career-highlights.html (flipping `locked` to `unlocked` on the gated-content div). But the real work was conceptual: fundamentally rethinking what the career-highlights page is for and how it should be structured.

This is a build journal entry about *not* building — about the strategic work that has to happen before a single line of code makes sense.

---

## The Problem with the Current Page

The existing career-highlights.html is a vertical timeline with 30+ initiative cards organized by company and role. Each card follows a problem/bet/shipped/outcome structure. The content is thorough and well-organized. It's also counterproductive.

The page activates multiple negative heuristics simultaneously:
- **Volume triggers disbelief.** 30+ initiatives from one person over ~13 years makes recruiters assume inflation or shallow involvement.
- **Breadth triggers "jack of all trades."** Auth, payments, AI, mobile, marketplace, engagement — listing all of them suggests mastery of none.
- **Company names trigger banking bias.** Wells Fargo and First Republic dominate the timeline, activating "enterprise = slow" assumptions.
- **Chronological organization invites timeline scrutiny.** Any gaps, overlaps, or returns to the same employer get noticed and judged.

The access gate was Kiran's previous attempt to mitigate this — don't show everything to everyone, let people self-identify first. The gate came off today because the new approach will solve the problem differently.

---

## The New Concept: Four Zones, Not a Timeline

The session landed on a structure organized by *what Kiran brings* rather than *where he's been*:

**Zone 1: Scale** — Proof of operating at a level most PMs don't reach. Headline: "Scale most PMs never touch." Four metrics: 32M MAU, 27.5M AI interactions, 25M+ auth users, #9→#3 JD Power. The contextualizing line makes clear these are high-stakes (people's money and trust), not just high-volume.

**Zone 2: Range** — The compound advantage of owning the full digital surface. Domain tags (Auth → Payments → Growth → AI → Mobile → Marketplace → Engagement) connected with arrows to show they inform each other. The reframe: breadth isn't scattered, it's compounding.

**Zone 3: AI Trajectory** — A focused progression from 2016 (document extraction) through 2025 (LLM-native product strategy). Time is used *locally within the zone* to show progression, but the page itself isn't organized by time. This is the "not jumping on a trend" proof point.

**Zone 4: Builder** — Points outward to the site itself, Fenix, MadLab, and the teardowns. The proof of hands-on capability is the artifact you're standing inside.

### Why This Structure Works

Company names are absent or minimal. Dates appear only in the AI trajectory (where they prove longevity). The organizing principle is capability, not employment history. A recruiter can't trigger banking bias if the banks aren't named. They can't doubt depth if each zone focuses on one dimension with concentrated proof.

### What Still Needs Work

The four-zone content is drafted but not locked. Key decisions remaining:
- Exact copy for each zone (headlines and descriptions are at ~80%, metrics need verification)
- "Digital experience" terminology was flagged as dated — settled on "end-to-end user journey" instead
- The mockup HTML exists but needs visual identity work (icons, path design) — Claude fingerprint risk is high on the illustrative elements
- How the page connects to the "second act" (persona-identified curated experience) is still open

---

## Technical Artifact: The Mockup

A working HTML mockup was created (`career-infographic-mockup.html`) demonstrating:
- Four spatial zones with alternating left/right layout
- Winding SVG path drawn dynamically based on zone positions
- Scroll-driven path animation (path draws as you scroll)
- Scroll-based reveal animations per zone and per metric badge
- Dark/light theme support using the site's CSS custom properties
- Responsive behavior (zones collapse to left-aligned on mobile)

The mockup is structural proof-of-concept, not production-ready. It proves the layout works but the visual identity needs external asset generation (custom icon set, potentially a hand-designed path shape).

---

## The Naming Decision

"Career Highlights" was retired as the page name. The new name: **"Track Record."**

The reasoning was grounded in recruiter psychology: a hiring manager scanning the nav is asking "where do I see what this person has actually done?" — and "Track Record" is literally the vocabulary they use when evaluating candidates. It's zero-ambiguity, it promises results and evidence, and it doesn't set format expectations that the infographic would violate.

The "My" possessive was considered and dropped — shorter is stronger in a nav, and the possessive is implied on a personal site.
