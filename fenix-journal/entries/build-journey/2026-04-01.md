# The Build Journey — From Prototype to Production
**April 1, 2026**

---

## Two Builds in One Session

This session had two distinct build phases. The first was content architecture: executing the five-phase trimming gameplan to take the Track Record prototype from 26 pairs to 16 polished cards with domain pills, calibrated descriptions, and a refined synthesis. The second was production engineering: converting the prototype into the-work.html, a deployed page on kiranrao.ai with full theme support, responsive layout, scroll animations, OG image, and live deployment.

---

## Phase 1: Content Architecture (v5 → v6)

### The Trimming

26 pairs → 16 was the target. The actual cuts:

- **Era 1** (8→5): Cut DailyChange (no metric, dated gamification framing), Alerts Next Gen (infrastructure, no number). Bundled Apple Watch into Mobile App Launch.
- **Era 2** (9→5): The hardest cut. Merged Cloud + Platform Migration into "Infrastructure Modernization" with +18% revenue metric. Cut Apple Pay (every bank shipped it), Digital Wealth (bundled into Eagle Invest), Digital Lending (absorbed into COVID).
- **Era 3** (8→5): Cut Teen Banking (no metric), Open Banking (infrastructure story). Bundled AI Insights into Fargo AI (3.1x metric folded in). Kept Segmentation standalone for its +19% metric.
- **Era 4** (1→1): Avatour stays.

The trimming decisions were fast — Kiran approved the entire cut/keep/bundle matrix in one pass. The bias pressure tests (Phase 0) did the heavy lifting by identifying Era 2 as the banking stigma and scroll fatigue epicenter. That diagnostic framing made the cuts feel obvious rather than painful.

### Domain Pills: Market-Relevant Taxonomy

Seven pill types: AI/ML (amber), Growth (green), Identity & Fraud (blue), Mobile (purple), Platform (muted), Personalization (warm), Payments (teal). The key insight was labeling cards with *what hiring managers search for* rather than internal banking categories. The AI/ML pill appearing across all four eras visualizes the 10-year AI thread without claiming it in text.

The pill taxonomy went through one revision: "Platform" → "Mobile" on three cards where the work was consumer app development, not infrastructure. Infrastructure Modernization kept "Platform" as the one accurate use. This was a good catch from Kiran — "does platform land better than mobile?" — and the answer was no.

### Card Calibration: One-Liner Formula

Phase 3 shortened every card description to one line. The principle: the metric is the hook, the description gives just enough context to make it believable, but not enough to answer "how did you do that?" The privacy note at the top ("the full picture is a conversation worth having") creates permission for this restraint.

Cards that survived unchanged: Cardless ATM ("users chose it even when they had their card" — too good to cut), JD Power (staccato phrasing works).

## Phase 2: Production Build (the-work.html)

### Light Mode — Two Missing Pieces

The page had no light mode support. Two causes: (1) No `[data-theme="light"]` CSS overrides for the page's custom variables, and (2) no theme toggle JavaScript (the page's inline scripts only had mobile menu and share logic).

Added 24 light-mode CSS rules covering every custom variable and component. The domain pills needed color darkening for contrast — green #7bc47f → #3a7d3e, blue → #3a6d8e, etc. Theme toggle JS was added inline (not via app.js, which would have created duplicate event listeners for shared nav components).

### The Hero Whitespace Bug

Shared styles.css has `.hero { min-height: 100vh }` which forced the hero section to fill the entire viewport. Fix: `min-height: auto; display: block;` override. Initial padding-top of 24px put the title behind the fixed nav — corrected to 72px.

### GSAP ScrollTrigger Failure → CSS Replacement

This was the session's most educational technical problem. GSAP's `once: true` ScrollTrigger property didn't work as a one-shot play trigger — it created scrub-style animation tied to scroll progress. Replacing with `toggleActions: 'play none none none'` also failed.

The fix was radical: rip out GSAP entirely and replace with pure CSS transitions + IntersectionObserver. `.reveal` class with opacity/transform transitions, direction variants (`.from-left`, `.from-right`, `.scale-in`), and an observer with `rootMargin: '0px 0px -15% 0px'` that unobserves after first intersection. This also eliminated two external script loads (gsap.min.js + ScrollTrigger.min.js), improving page weight.

The lesson: sometimes the framework is the problem. GSAP ScrollTrigger's API surface area made it hard to get simple "animate once on enter" behavior. A 15-line IntersectionObserver + CSS transitions did the same thing more reliably.

### Timeline Center Line Bleed

The `.timeline::before` pseudo-element (2px amber vertical line) was visible behind era header text and transition beat text. Fix: `z-index: 2; background: var(--bg);` on `.era-header` and `.transition-beat`. Simple layering, but the visual artifact was immediately noticeable.

### OG Image

Generated using Pillow — cropped the veteran-hero-2-1.png (bear character with city skyline), resized to 1200x630, added semi-transparent glass overlay with "15+ YEARS BUILDING / THE WORK / Enterprise to startup. The full arc." matching the homepage bento card aesthetic. The bento-cards.js was updated from "Career Highlights" → "The Work" for the homepage card.

## Technical Debt Identified

The page uses entirely inline CSS and JS (no external stylesheet or script beyond shared styles.css and fenix-widget). At ~1000 lines of inline CSS, this is getting heavy. A future refactor could extract to the-work.css, but for now the single-file approach avoids cache coordination issues.
