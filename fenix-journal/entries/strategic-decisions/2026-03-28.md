# Strategic Decisions — March 28, 2026

---

### Short Copy Is Better Copy: Mobile Constraint as Universal Improvement
**Date:** 2026-03-28
**Category:** Product Philosophy

**Decision:** Shortened card descriptions created for mobile readability replaced the longer desktop versions everywhere. Short copy is now the only copy — no device-specific text variants.

**Context:** Mobile overlay compression required tighter descriptions to fit one line without truncation. Initial approach was a `mobileCopy` object that swapped descriptions on small screens, maintaining longer desktop versions. After seeing the shortened versions, Kiran immediately said "make it universal everywhere. Short is better."

**Reasoning:** The bento card descriptions are scannable labels, not paragraphs. "Popular app teardowns. Keep, kill, rebuild." communicates more efficiently than a full sentence. The mobile constraint forced better writing. Maintaining two versions adds complexity with no benefit — if the short version is better, it's better everywhere.

**Impact:** Eliminates a code path (`mobileCopy` object + swap logic), simplifies `cardData`, and establishes a principle: when a responsive constraint produces better content, propagate it universally rather than maintaining per-breakpoint variants.

**Open Questions:** None.

### Mobile/Tablet: Locked and Deprioritized
**Date:** 2026-03-28
**Category:** Architecture

**Decision:** Mobile and tablet responsive behavior for the bento grid is locked. No further CSS or layout work until all desktop images are in place. The 14 remaining MJ images are desktop-only concerns — mobile uses a fixed set of per-card images regardless of persona.

**Context:** The session completed mobile image overrides (all 9 cards guaranteed an image at ≤1024px), overlay compression, copy shortening, and morph choreography wiring. Meanwhile, desktop has 22 gradient-fallback slots that need MJ art. Kiran ran out of MJ credits, so the image generation is paused.

**Reasoning:** Mobile is the higher-traffic viewport but also the simpler layout (single column, fixed images). Desktop is where the persona-specific card rotation creates visual variety — and where missing images are most noticeable as gradient placeholders. Locking mobile means no regression risk while iterating on desktop art.

**Impact:** `mobileImageOverrides` is the stable contract for mobile. Future desktop image additions only update `imageMap` — they don't touch mobile behavior. This separation of concerns between mobile (per-card) and desktop (per-card-per-slot) image strategies simplifies the remaining work.

**Open Questions:** None.
