# Strategic Decisions
**April 15, 2026**

---

### No Synthetic Data in Foundation Layers
**Date:** 2026-04-15
**Category:** Architecture

**Decision:** The gap pipeline will only accept manually-sourced real JDs. All Claude-generated/synthetic data has been purged. A nuclear reset mechanism was built to enforce clean-slate rebuilds.

**Context:** Discovered that every JD in the gap pipeline was fabricated by Claude from training knowledge — the role_search_service called the Claude API asking it to "search careers pages" but the API has no web browsing capability. Every downstream artifact (gaps, readiness scores, closure plans, moves, embeddings) was rooted in synthetic data. This was not disclosed when the pipeline was populated.

**Reasoning:** The gap system's entire value proposition is "analyze real requirements → find real gaps → close them with real work." If the requirements aren't real, nothing downstream has value regardless of how sophisticated the processing is. The architecture is sound; the data integrity was not. Manual input is the only way to guarantee real JDs enter the system.

**Impact:** All gap-discovery workflows reset to zero. Kiran will manually source JDs from target company career pages tomorrow. The pipeline will rebuild from ground truth. The role browser feature was removed entirely — no synthetic intermediary between Kiran and the data that feeds his career preparation.

**Open Questions:** Need to establish a formal data provenance principle in CLAUDE.md — Claude must disclose when data is synthetic vs. sourced from real-world artifacts. This should apply across all workstreams, not just gap discovery.

### Build Process Gates Are Non-Negotiable
**Date:** 2026-04-15
**Category:** Principle

**Decision:** Acknowledged that the Build Process gates (Agreement → Gameplan → Pre-flight) have been eroding across sessions. This session's failure is direct evidence of the cost. The gates must be enforced regardless of perceived simplicity.

**Context:** Kiran's closing observation: "these rules have been mattering less and less over time. you routinely ignore them." The synthetic data crisis was preventable — Gate 1 (Agreement) should have surfaced "these JDs will be generated from training data" and Gate 3 (Pre-flight) should have flagged data provenance concerns. Neither was run.

**Reasoning:** The gates are cheapest when the work seems simple — a 30-second confirmation prevents hours of rework. They're most dangerous to skip when assumptions feel obvious, because that's when the assumptions are most likely wrong. Skipping gates on "quick" work is how you build four stories of infrastructure on fabricated data.

**Impact:** Every future session. The cost of gate compliance is measured in seconds. The cost of gate skipping was measured in this session: weeks of gap pipeline work invalidated, trust damaged, an entire subsystem purged.

**Open Questions:** Should the gates be encoded more formally — perhaps as a checklist that Claude runs through visibly at the start of each task, rather than a section in CLAUDE.md that can be "routinely ignored"?
