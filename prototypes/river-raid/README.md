# River Raid prototype

GW-BASIC River Raid clone for the CTO/Technologist persona.

## What's here

```
prototypes/river-raid/
├── index.html              # The prototype page (card → modal → game)
├── build-bundle.sh         # Zips the bundle for js-dos
├── README.md               # This file
└── bundle/
    ├── RIVER.BAS           # The game source (real GW-BASIC, ~250 lines)
    ├── dosbox.conf         # DOSBox config + autoexec
    ├── GWBASIC.EXE         # ⚠️  YOU PROVIDE THIS — see below
    └── riverraid.jsdos     # ⚠️  Generated by build-bundle.sh
```

## What you need to do

### 1. Drop GWBASIC.EXE into `bundle/`

I can't pull Microsoft binaries from the sandbox, so you grab it once.

**Easiest source:** Internet Archive hosts a freely downloadable copy:
https://archive.org/details/GW-BASIC_3.23

Download the disk image, extract `GWBASIC.EXE`, and drop it in `bundle/`.

It's a ~70KB file. Microsoft has been quiet about redistribution for 40 years and the Internet Archive ships it openly, so practically this is fine. If you want zero risk, we can swap in PC-BASIC (open-source clone) later — but that's a different conversation.

### 2. Build the bundle

```bash
cd prototypes/river-raid
chmod +x build-bundle.sh
./build-bundle.sh
```

This creates `bundle/riverraid.jsdos` (a zip with dosbox.conf + GWBASIC.EXE + RIVER.BAS).

### 3. Open the prototype

You can't just double-click `index.html` — js-dos needs to fetch the bundle over HTTP, not from `file://`. Run a tiny static server:

```bash
cd prototypes/river-raid
python3 -m http.server 8080
```

Then open http://localhost:8080/

### 4. Play

- Card on the left = the prototype card. Click it.
- Modal opens. Click "START ENGINES".
- DOSBox boots, GW-BASIC loads, RIVER.BAS runs.
- Arrows steer, space fires, dodge banks/enemies, refuel on magenta depots.
- **Easter egg:** Ctrl+Break drops to the `Ok` prompt → type `LIST` → read 1992-vintage source.
