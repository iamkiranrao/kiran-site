How this works

Architecture in one line:

browser → POST /api/scan → Vercel function → mode-specific prompt → Gemini 2.5 Flash → structured JSON → you

Mode-specific prompt chains. The mode you pick (Menu / Grocery / Produce / Supplements / Skincare / Pet / Recipe / Grow / Beverage) routes the image through a different system prompt — nine in total. Each one is tuned for the visual domain: the Menu prompt knows about hidden allergens in cuisine families and religious dietary law; the Pet prompt knows what's toxic to dogs vs. cats; the Skincare prompt watches for irritants and comedogenic ingredients. Same image, different mode, different analysis.

Structured output. The model is constrained to return JSON with a fixed schema: a verdict section (the butler's 2-3 sentence summary), domain-specific sections with risk-tagged items, and an overallRiskLevel of safe, caution, warning, or danger. The shape doesn't drift between modes — the renderer above doesn't have to know which mode it's displaying.

Real-time on Vercel. A single serverless function. Cold-starts in under 500ms, processes a scan end-to-end in 1.5–3 seconds. Rate-limited at 50 scans/day per device during beta. The Gemini call streams back as JSON, gets parsed and returned.
